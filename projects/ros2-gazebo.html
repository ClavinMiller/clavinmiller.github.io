项目背景

机器人项目在接入真实硬件前，需要一个可以低成本重复实验的环境。这个项目的重点不是完成一次安装，而是建立清晰、可复现的工作空间：版本对应明确、依赖来源可追踪、基础通信和仿真模型能够被验证。

实现过程

  1. 安装系统更新、区域设置和 ROS2 软件源，确认发行版与 Ubuntu 版本对应。
  2. 创建 colcon 工作空间，完成 publisher / subscriber 最小通信验证。
  3. 安装仿真桥接包，启动空世界并检查渲染与物理引擎。
  4. 加入基础 URDF / Xacro 模型、关节和传感器插件。
  5. 整理 setup 脚本、依赖清单与启动命令，保证新终端可重复加载。

问题与解决方案

软件包版本或教程不对应

以官方发行版文档为主，并在笔记中固定 Ubuntu、ROS2 与 Gazebo 的具体组合，避免混用不同年代的命令。

模型或 mesh 无法加载

逐项检查资源 URI、包安装路径和环境变量；先用最小几何体确认模型结构，再恢复外部资源。

工作空间 source 顺序混乱

明确系统环境和 overlay workspace 的加载顺序，不把失效路径长期写进 shell 配置。

复现要求每次修复都记录“症状、原因、验证命令”，而不仅留下最终可运行的一串命令。
